Overview

The ActiveRaUL system consists of two main parts:
  • RESTful Web service that provides a uniform way to manage widgets and the data that is processed by the widgets. It supports CRUD operations, such as storing and querying RDF data, on the underlying data store via HTTP requests. Additionally, the service provides a mechanism to produce and render widgets via RDFa annotated XHTML forms.

  • Client-side JavaScript RDFa API based on jQuery and enables the direct interaction between the user (or more precisely the generated widgets) and the Ac- tiveRaUL service. The API supports RDFa parsing of the annotated XHTML forms and encapsulates the communication with the backend service. A major aspect of client-side processing is the transparent databinding of values stored in the backend or in the Linked Data Web to controls comprised in the widgets. This includes up- dating the RDF triples in the backend and pre-populating existing forms if data is available.

RaUL Widget elements

RaUL defines a widget ontology that supports all HTML form elements. Example widget elements are:
  • raul:Textbox
  • raul:Group
  • raul:Listbox
The full ontology can be found at: http://purl.org/NET/raul#.

Demo

Our use case simulates a second-hand goods trading platform at which registered users can sell and buy second-hand goods. For demonstration purposes, and later for understanding the value provided by our approach, we consider a scenario in which a typical user performs three basic (yet common) operations: registration, selling a product and buying a product.

  1. User Registration. A new seller/buyer needs to register for a new user account to act on the e-Commerce website. The user requests the registration form, which offers him the standard fields to fill in his user data. One option would be to directly use the FOAF profile.
  2. Product Advertisement. A user, once registered, can add products for selling. The associated form / widget consists of examples of common fields used to describe a product. Similar to the first operation, the user could reuse an existing description of the product on the Linked Data Web, by entering the product's URI.
  3. Bid on a Product. Finally, the user may want to purchase an advertised product. The product can be easily found by filling in the form / widget. The user can then bid for the product.
